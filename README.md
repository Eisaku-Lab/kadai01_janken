# ①課題名
カイジの限定じゃんけん（簡易版）

## ②課題内容（どんな作品か）
漫画「賭博黙示録カイジ」に登場する“限定じゃんけん“を簡易に再現したゲーム。
【ルール】
・プレイヤーとコンピュータは各手（グー・チョキ・パー）を2回ずつ使用可能
・双方が★を2つ保有してスタート
・じゃんけんに負けた方は★を1つ失う
・★が0になった方の敗北

  ※実際のURLに置き換えてください

## ⑤工夫した点・こだわった点
### アプリの制作意図
- 元々のじゃんけん課題にゲーム性を加えてよりおもしろくする
 - 実際のじゃんけんと比較して不足していた要素
 - ① 報酬性：勝敗に意味を持たせる仕組み
 - ② 連続性：過去の選択と結果が次の選択に影響を与える構造
 - これらを実現するルールで、デジタル表現に適した「カイジの限定じゃんけん」を採用

### 工夫した点
- 配列を活用した手札管理
 - let playercard = [0, 2, 2, 2] // let computercard = [0, 2, 2, 2]
 - 配列を使用して各手の残り回数を管理
 - 使用するたびに配列の値を-1することでカウントダウンを実現
- AND条件を使った複合条件分岐
 - if (computercard[1] >= 1 && r == 1) // カードが残っているかつその手が選ばれた場合のみ実行
 - カードが0枚の手は自動的に選択肢から除外される仕組み

## ⑥難しかった点・次回トライしたいこと（又は機能）
### 難しかった点
- 手札管理
 - メンターさんに要件を伝えたら“配列“というヒントをもらい、実装方法を調べてどうにか実装
### 次回トライしたいこと
- 手札残数0後のコンピュータの動作改善
 - 現状の課題：コンピュータが手札残数0の手を選ぼうとすると「何も起こらない試行」が発生
 - 改善方針：残っているカードのみから自動選択する仕組みの実装
- アニメーション・見た目の改善
